How NewsLens Works

A 6-module autonomous pipeline that ingests live news, routes queries through specialist agents, and surfaces multi-perspective analysis with full reasoning transparency.

📡

M0 — Live Ingestion Pipeline

Pathway Streaming

Continuously ingests from NewsAPI and RSS feeds using Pathway's incremental computation engine. Articles are chunked (512 tokens / 64 overlap), embedded, and indexed into a live vector store that auto-refreshes without manual intervention.

🧠

M1 — Intent Classifier

Classifies raw queries into one of three intents: TIMELINE, BIAS_DETECTION, or CROSS_PUBLISHER_SUMMARY. Extracts entities, publishers, and date ranges. Falls back to regex classification when no LLM key is available.

🤖

M2 — Multi-Agent Router (LangGraph)

A stateful LangGraph graph: Supervisor → Retrieve → CRAG Evaluate → Specialist Agent → Validate → Assemble. CRAG filters retrieved chunks by relevance, escalating through Pathway → Bing → Scraper fallback tiers automatically.

⚖️

M3 — Bias & Sentiment Engine

Computes per-publisher sentiment (VADER / RoBERTa), 5-dimensional framing vectors (Conflict / Economic / Human Interest / Morality / Responsibility), entity salience deltas, and a normalised bias score in [-1, +1].

🕐

M4 — Timeline Synthesizer

Extracts temporally anchored events via spaCy NER + LLM secondary pass. Deduplicates using Jaccard similarity, detects coverage gaps, computes narrative coherence scores, and assigns HIGH/MEDIUM/LOW/UNVERIFIED confidence based on source count.

🖥️

M5 — Explanation & UI Engine

This interface. A FastAPI + Jinja2 + Vanilla JS frontend with SSE live progress streaming, full agent trace panel, retrieval-tier badges, metadata card, Timeline-first 4-tab results, and Chart.js sentiment + framing visualisations.
